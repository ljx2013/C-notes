
# C++ 异常处理

## 一、异常处理概述

异常处理是一种处理程序运行时错误的机制，允许程序在发生错误时优雅地恢复或报告错误。

### 1.1 异常处理的组成

- **try**：标识可能抛出异常的代码块
- **catch**：捕获并处理异常
- **throw**：抛出异常

## 二、基本语法

### 2.1 try-catch 结构

```cpp
try {
    // 可能抛出异常的代码
    if (error_condition) {
        throw exception_object;
    }
} catch (exception_type1 e) {
    // 处理类型1的异常
} catch (exception_type2 e) {
    // 处理类型2的异常
}
```

### 2.2 示例：基本异常处理

```cpp
#include <iostream>
using namespace std;

int divide(int a, int b) {
    if (b == 0) {
        throw "Division by zero!";
    }
    return a / b;
}

int main() {
    try {
        int result = divide(10, 0);
        cout << "Result: " << result << endl;
    } catch (const char* msg) {
        cout << "Error: " << msg << endl;
    }
    return 0;
}
```

## 三、标准异常类

### 3.1 异常类层次

```
exception
├── bad_alloc
├── bad_cast
├── bad_typeid
├── bad_exception
└── logic_error
    ├── domain_error
    ├── invalid_argument
    ├── length_error
    └── out_of_range
└── runtime_error
    ├── range_error
    ├── overflow_error
    └── underflow_error
```

### 3.2 使用标准异常

```cpp
#include <iostream>
#include <stdexcept>
using namespace std;

void validateAge(int age) {
    if (age < 0) {
        throw invalid_argument("Age cannot be negative");
    }
    if (age > 120) {
        throw out_of_range("Age out of valid range");
    }
}

int main() {
    try {
        validateAge(-5);
    } catch (const invalid_argument& e) {
        cout << "Invalid argument: " << e.what() << endl;
    } catch (const out_of_range& e) {
        cout << "Out of range: " << e.what() << endl;
    }
    return 0;
}
```

## 四、自定义异常类

### 4.1 创建自定义异常

```cpp
#include <exception>
#include <string>
using namespace std;

class MyException : public exception {
private:
    string message;
public:
    MyException(const string& msg) : message(msg) {}
    
    const char* what() const noexcept override {
        return message.c_str();
    }
};

void doSomething() {
    throw MyException("Something went wrong");
}
```

### 4.2 多层异常继承

```cpp
class NetworkException : public exception {
protected:
    string message;
public:
    NetworkException(const string& msg) : message(msg) {}
    const char* what() const noexcept override {
        return message.c_str();
    }
};

class ConnectionException : public NetworkException {
public:
    ConnectionException(const string& msg) : NetworkException(msg) {}
};
```

## 五、异常处理的高级特性

### 5.1 捕获所有异常

```cpp
try {
    // 可能抛出各种异常
} catch (...) {
    cout << "Unknown exception caught" << endl;
}
```

### 5.2 异常重新抛出

```cpp
void process() {
    try {
        // 代码
    } catch (const exception& e) {
        // 部分处理
        throw;  // 重新抛出原异常
    }
}
```

### 5.3 noexcept 修饰符

```cpp
void func() noexcept {
    // 承诺不抛出异常
}

void func() noexcept(false) {
    // 可能抛出异常（默认）
}
```

## 六、异常安全

### 6.1 RAII 原则

资源获取即初始化，利用析构函数保证资源释放。

```cpp
class FileHandler {
private:
    FILE* file;
public:
    FileHandler(const char* filename) {
        file = fopen(filename, "r");
        if (!file) {
            throw runtime_error("Failed to open file");
        }
    }
    
    ~FileHandler() {
        if (file) {
            fclose(file);  // 保证资源释放
        }
    }
};
```

### 6.2 异常安全级别

| 级别 | 保证 |
|------|------|
| no-throw | 保证不抛异常 |
| strong | 异常发生时状态不变 |
| basic | 不泄漏资源 |

## 七、异常处理的最佳实践

### 7.1 何时使用异常

- 处理不可预期的错误
- 错误发生在子系统边界
- 需要跨函数传递错误信息

### 7.2 何时不使用异常

- 性能关键代码（异常有开销）
- 简单的参数校验（使用返回值）
- 编译时错误（使用静态断言）

### 7.3 异常处理指导原则

1. 只在异常情况下使用异常
2. 提供有意义的错误信息
3. 避免异常泄露资源
4. 使用 RAII 管理资源
5. 不要在析构函数中抛出异常

## 八、实战示例

### 示例1：异常处理的完整示例

```cpp
#include <iostream>
#include <stdexcept>
#include <vector>
using namespace std;

int getElement(const vector<int>& vec, int index) {
    if (index < 0 || index >= vec.size()) {
        throw out_of_range("Index out of bounds");
    }
    return vec[index];
}

int main() {
    vector<int> nums = {1, 2, 3, 4, 5};
    
    try {
        cout << getElement(nums, 2) << endl;
        cout << getElement(nums, 10) << endl;
    } catch (const out_of_range& e) {
        cout << "Error: " << e.what() << endl;
    }
    
    cout << "Program continues..." << endl;
    return 0;
}
```

### 示例2：异常与资源管理

```cpp
#include <iostream>
#include <memory>
#include <stdexcept>
using namespace std;

class Database {
public:
    Database() {
        cout << "Connecting to database..." << endl;
    }
    
    ~Database() {
        cout << "Disconnecting from database..." << endl;
    }
    
    void query() {
        throw runtime_error("Query failed");
    }
};

int main() {
    try {
        unique_ptr<Database> db = make_unique<Database>();
        db->query();
    } catch (const exception& e) {
        cout << "Error: " << e.what() << endl;
    }
    // db 自动销毁，资源正确释放
    return 0;
}
```

## 九、总结

| 概念 | 说明 |
|------|------|
| try | 标识可能抛异常的代码块 |

| throw | 抛出异常对象 |

| 自定义异常 | 继承 exception 类 |

异常处理是编写健壮程序的重要工具，合理使用可以显著提升代码的可靠性和可维护性。
